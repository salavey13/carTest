## What the fuck is SupaPlan?

SupaPlan is the coordination brain that lets AI agents (like Codex) work on your repo while you do literally nothing.

It turns your GitHub repo into a self-growing garden:
- Supabase = the task list (what needs to be done)
- GitHub = the code (where the work actually happens)
- Codex = the gardener (does the actual work)

You no longer write todo lists.  
You just say **“ебаш!”** and walk away.

## The 4-Step Dummy Guide (literally 4 steps)

### Step 1: Fork / Join the Team (one-time, 30 seconds)
You already did this.  
Your repo is already connected to Codex.  
If not — just click the fork button on GitHub and give Codex access. Done.

### Step 2: Give Codex the Magic Command
Open any chat with Codex (Telegram, Slack, or this very thread) and type exactly:

**“ебаш!”**

That’s it.  

Codex will:
- Look at SupaPlan (the task list in Supabase)
- Pick the next open task it can do
- Claim it (so no one else touches it)
- Do the work
- Open a small PR when ready

You can go make coffee.

### Step 3: Sit Back and Enjoy
While you’re taking a shit, scrolling memes, or sleeping — Codex is working.

It will keep saying things like:
- “Claimed task #42 — adding pH for dummies”
- “Ready for PR — check the link”
- “Next task picked automatically”

You don’t have to do anything until you see a PR.

### Step 4: Salavey13 Merges (your only job)
When you see a PR with `supaplan_task:123` in the title — just click **Merge**.

That’s it.  
The GitHub Action automatically marks the task as **done** in SupaPlan.

Codex immediately picks the next task (even if the previous one isn’t merged yet — unless there’s an explicit dependency).

## Bonus: Task Dependencies (built-in safety)

If a task needs something else first (e.g. “add pH page” depends on “create academy folder”), we mark it in SupaPlan.

Codex will automatically:
- Check dependencies before claiming
- Wait (or skip) until the blocking task is merged

You never have to think about order. Codex does.
