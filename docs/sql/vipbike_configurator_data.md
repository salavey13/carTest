## Configurator Insights & Recommendations

### User Flow for Configurator

1. **Step 1: Model Selection**
   - Display all motorcycle models as cards with images
   - Show key specs: power, max speed, base price
   - Allow filtering by price range and type (sport, budget, premium)

2. **Step 2: Battery Selection**
   - After selecting a model, show battery options
   - Display estimated range for each battery capacity
   - Show price difference (incremental cost)
   - Option to choose between Regular and Lithium batteries

3. **Step 3: Additional Options (if applicable)**
   - Warranty extension
   - Delivery options
   - Accessories

4. **Step 4: Summary & Contact**
   - Final configuration summary
   - Total price calculation
   - Contact form or direct link to Telegram

### Key Configurator Features

#### Battery Calculator
```
Base Price + Battery Price = Total Price
Example: G8 (120,800₽) + 80Ah Lithium (98,000₽) = 218,800₽
```

#### Range Estimator
Range varies based on:
- Battery capacity (primary factor)
- Riding speed (higher speed = less range)
- Terrain (flat vs hills)
- Temperature (cold reduces range)

## Database Schema Mapping

For the `public.cars` table, here's how to map the PDF data:

| PDF Field | Database Field | Notes |
|-----------|----------------|-------|
| Model Name | `model` | e.g., "G8", "R6", etc. |
| Manufacturer | `make` | "VipBike" for all |
| - | `type` | "bike" for motorcycles, "parts" for batteries |
| Base Price | `daily_price` | Store base price in kopecks or as decimal |
| Specs (power, speed) | `specs` JSONB | Store as JSON object |
| Description | `description` | Generate from specs and model info |
| Image | `image_url` | To be populated with actual images |
| Battery options | `specs.battery_options` | Store pricing matrix |
