# Питч для Алисы: GREENBOX × PROJECT TESSERACT

Алиса, привет! 💚

Мы собрали для тебя готовый план, как превратить идею умной теплицы в рабочую систему без боли и перегруза по железу на старте.

## Что предлагаем
- Делаем **simulator-first**: сначала оттачиваем UI/UX, сценарии и алерты на тестовых данных.
- Строим всё как **игровой production loop**: уровни-сценарии, инциденты, очки стабильности, повторы (replay).
- Ведём работу в **одном общем плане** (`docs/GREENBOX_TESSERACT_GAMELAB_PLAN.md`), который обновляют и люди, и Codex.
- Используем **multi-tenant структуру по slug**, чтобы у каждой команды/теплицы были свои настройки, статистика и правила.

## Почему это выгодно
- Быстрое внедрение без блокера "сначала подключим всё железо".
- Понятный онбординг для новичков (есть разделы "для чайников").
- Меньше риска ошибок при реальном запуске, потому что критические сценарии прогоняются заранее.
- Легко масштабировать: от 2 теплиц до "кубической" модели Tesseract.

## Что уже заложено в план
- Структура страниц (`/greenbox/[slug]/*`),
- academy-блок по датчикам и автоматике простым языком,
- сценарный движок (levels + timeline + expected actions),
- дорожная карта G0–G6 с пошаговым запуском.
