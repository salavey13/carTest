# Как сгенерить договор аренды электромотоцикла

## Способ 1: Веб-интерфейс (рекомендуется, 2026-06)

**Для арендаторов и владельцев экипажей — прямо в Telegram WebApp.**

### Как это работает (новый flow)

1. **Арендатор заполняет форму** — в WebApp на странице аренды или в `ContractDraftPanel`
   - ФИО, паспорт, телефон, email
   - Выданное оборудование (ключи, шлемы, зарядка)
   - Состояние при выдаче (пробег, заряд/топливо, повреждения)

2. **Черновик сохраняется** — данные уходят в `rental.metadata.contract_draft`
   - Статус: `pending`
   - В Telegram владельцу экипажа летит уведомление с inline-кнопками

3. **Владелец утверждает/отклоняет**
   - Нажимает "Утвердить" → генерируется DOCX
   - Нажимает "Отклонить" → арендатор получает уведомление

4. **Генерация DOCX** — (шаблон тот же, что и в старом flow)
   - HTML-шаблон заполняется данными из черновика
   - Конвертируется в DOCX через cheerio
   - Загружается в Supabase Storage (`rental-contracts` bucket)

5. **Доставка** — DOCX отправляется обоим сторонам через Telegram Bot API
   - Арендатор получает договор
   - Владелец получает копию

6. **Верификация** — запись в `rental_contract_artifacts`
   - `contract_key` = уникальный ключ
   - `storage_path` = путь к файлу в Storage
   - SHA256-хesh для антиподделки

### QR-коды и deeplinks

Формат QR: `rent_{bikeId}_{docSha256}`
- **Арендатор сканирует** → попадает на страницу checkout (заполнение формы)
- **Владелец сканирует** → попадает на `/franchize/{slug}/contract-draft/{rentalId}`
  - Видит черновик, может утвердить/отклонить
  - После утверждения видит QR для клиента

### Таблицы БД

**`rentals.metadata`** — хранит черновик договора:
```json
{
  "contract_draft": {
    "status": "pending|approved|declined",
    "submitted_at": "2026-06-18T...",
    "submitted_by": 123456789,
    "renterData": { ... },
    "equipmentData": { ... },
    "pickupData": { ... }
  },
  "contract_key": "rental-{bikeId}-{timestamp}"
}
```

**`rental_contract_artifacts`** (private schema):
- `contract_key` — уникальный ключ договора
- `storage_path` — путь к DOCX в Supabase Storage
- `original_sha256` — хеш для верификации
- `created_at` — timestamp создания

## Способ 2: Codex-бот (legacy)

**Для быстрой генерации через Slack.**

Пишешь в Slack: **"@codex сделай документ falcon-gt с завтра 18:00 до 28.05 10:00"** — и в Телегу прилетает готовый DOCX-договор.

### Как это работает

1. **Запрос** — бот вытаскивает из фразы: какой мот и даты аренды
2. **Поиск мота** — скрипт ищет в Supabase (таблица `cars`, type=bike/ebike), подбирает лучший матч по fuzzy-score
3. **Данные арендатора** — берутся из OCR паспорта и ВУ (файлы JSON), которые бот получает заранее
4. **Заполнение шаблона** — HTML-шаблон договора заполняется ~40 переменными (mustache `{{variable}}`)
5. **Генерация DOCX** — заполненный HTML конвертируется в DOCX через cheerio
6. **Доставка** — DOCX отправляется в Телегу через Bot API
7. **Верификация** — хеш сохраняется в `rental_contract_artifacts`

Включается флагом `--saveMetadata 1` при запуске скрипта.

## Server Actions (новый flow)

- `submitContractDraft` — арендатор отправляет черновик
- `approveContract` — владелец утверждает → генерация DOCX
- `declineContract` — владелец отклоняет
- `checkRentalOwnershipForQr` — проверка владельца для QR-роутинга
