# Factory Global Rules — универсальные правила фабрики (контент + дизайн)

> Единый канон для ВСЕХ агентов content-dept-fil-ai (локально) и OpenCode-агентов на сервере.
> Каждый агент читает этот файл в ШАГ 0 / шаге «Контекст» ПЕРЕД работой. Применять ВСЕГДА, без исключений.
> Источник: глобальный слой правил владельца фабрики. Это выжимка-канон; детальные методологии — в профильных скиллах (ссылки внизу).

## ⛔ АБСОЛЮТНЫЕ ЗАПРЕТЫ (нарушение = критический баг)

### 1. Слово «голос» в маркетинговом смысле — ЗАПРЕЩЕНО
- НИКОГДА не писать «голос бренда / клиента / эксперта», «brand voice», «tone of voice», «в голосе X», «найти свой голос».
- ВСЕГДА: «стиль речи», «стиль бренда», «стиль подачи», «манера речи», «авторский стиль», «стилистика».
- Везде: чат, тексты, промпты агентам, КП, brand-файлы, имена файлов (не `voice-style.md` → `style-of-speech.md`; существующий файл не переименовывать).
- При цитировании транскрипта, где звучит «голос» — заменять на «стиль речи» в своих текстах.

### 2. Эмодзи в вёрстке — ЗАПРЕЩЕНО
- В любых HTML/CSS/лендингах/КП/презентациях/карточках/баннерах — НЕ использовать эмодзи как иконки/декор.
- Только: inline-SVG, иконочные шрифты (Heroicons, Phosphor, Lucide, Tabler), CSS/SVG-буллеты.
- Эмодзи в ТЕКСТЕ поста соцсетей — допустимо по стилю клиента; речь только про вёрстку.

### 3. Картинки в клиентском HTML — ВШИВАТЬ в base64
- Любой HTML для клиента (КП, кейс, отчёт, дашборд, лендинг-документ) — растровые картинки как `data:image/...;base64`, не ссылкой на `assets/`.
- Сначала оптимизировать: ресайз до ≤1000–1400px, JPEG q80–85. Не вшивать сырые 4K.
- Иконки/логотип — inline SVG. Проверка перед сдачей: `grep -c 'src="assets/'` = 0.

### 4. Не выдумывать данные
- Цены, тарифы, цифры, кейсы — ТОЛЬКО из brand-файлов клиента. Нет данных → плейсхолдер `{{ЦЕНА}}`, спросить.
- Кейсы и цифры брать только из `brand/profile.md` / `brand/social-proof.md`.

## КОНТЕНТ — обязательные правила

- **Human-style обязателен.** Любой текст проходит гейт `human-style-check` (скилл) — убрать ИИ-штампы (17 категорий). Score <50 = reject, 50–70 = revise, >70 = pass. Без исключений — даже офферы, лендинги, стратегии.
- **Hook-first.** Первое предложение = хук ВСЕГДА.
- **Без воды.** Коротко, конкретика, без корпоративщины и канцелярита.
- **Стиль речи клиента** — из `brand/style-*.md` + `learning/corrections.md`. Применять ВСЕ коррекции клиента без исключения.
- **Платформенный тон** — из `brand/platform-strategy.md` (где «мы»/клиника, где «я»/личный бренд).
- **Реклама по ФЗ** (для мед/фин/юр ниш): не обещать результат, дисклеймеры — если в нише регулируется.

## ДИЗАЙН И ВИЗУАЛ — обязательные правила

- **Визуальные промпты — только через `visual-prompt-director`** (4 слоя: фон → персонаж → 3D-элементы → текст). Без него промпт не отправляется ни в один генератор.
- **Дефолт стиля документов = Neuro Alchemy** (`neuro-alchemy-docs` / `neuro-alchemy-presentation`), если стиль не задан явно или не указан бренд клиента.
- **Текст на слайдах — строго понятный русский.** Жаргон — только в развёрнутом тексте поста, не голым заголовком слайда.

### Генерация/перегенерация слайдов и картинок (gate-протокол)
- **Меняем ТОЛЬКО сломанное.** «Почини слайд» → менять лицо/одежду/опечатку, остальное (композиция, макет, цвета, шрифты) воспроизводить 1:1. Перед перегенерацией — открыть оригинал глазами, зафиксировать композицию (бюст / по пояс / full-body).
- **Гейт: один тест → показать → OK → батч.** Не жечь кредиты слепыми пачками.
- **Боевые файлы неприкосновенны до QA.** Генерить в `_test/`, в боевой путь — только после визуального QA + OK. Бэкап перед перезаписью.
- **FaceID — только по утверждённому референсу** (`image_input`), никогда text-to-image «по памяти». В промпте лицо: возраст+пол+одежда, без деталей внешности (генератор блокирует конкретных людей; черты — из референса).
- **Duotone съедает цвет:** не обещать конкретный цвет одежды в duotone — убрать объект из кадра или сменить палитру.
- **Вычитка кириллицы:** после генерации прочитать ВЕСЬ текст всех слайдов глазами (артефакты: «дёнь», дубль слога, мусор-слова, `0Р` вместо `0₽`). В промпте — `text exactly: "..."`.
- **Единство шрифта серии:** при перегенерации 1 слайда взять формулировку шрифта из ПЕРВОГО промпта серии, сверить вес/ширину с соседом. Для 100% единства — текст накладывать кодом, не AI-генерацией.
- **Правило 2 попыток:** одна проблема, та же формулировка 3-й раз → СТОП, причина системная, к пользователю с вариантами.

### Frontend / вёрстка (production-grade)
- **Никаких стен текста:** `<p>` >50 слов или 3+ предложения слитно — разбивать. Перечисления → `<ul>`.
- **Бренд жёстко:** палитра и 2 шрифта из style-guide клиента, не выходить за hex.
- **Mobile-QA перед «готово»:** проверить на 3 viewport (360/375/414) — нет горизонтального оверфлоу (`scrollW===docW`), вертикальный скролл достигает низа, hero не обрезает лицо, длинные CTA переносятся, тёмные секции — светлый текст.
- **Inline `style="grid-template-columns"` запрещён** — перебивает mobile-CSS (specificity 1000). Только классы-варианты.
- **Cache-bust на CSS:** версионировать `href="styles.css?v=YYYYMMDDx"` при каждом деплое (query на HTML кеш CSS не пробивает).
- **Проверка перед отчётом:** открыть артефакт глазами (desktop + mobile), не отчитываться «готово» без визуальной проверки.

## ОБЩИЕ ПРИНЦИПЫ РАБОТЫ

- **Точечные правки.** Не переписывать целиком. Менять минимально необходимое (стиль → только CSS; баг → только строки бага). Edit > Write если файл существует.
- **Не читать лишнее.** Только файлы, нужные для текущей задачи.
- **Verification.** Задача готова, когда: файл создан (путь указан), результат показан, гейт пройден. «Должно работать» без проверки = не готово.
- **Сохранять результаты только в папки проекта** (по карте из CLAUDE.md клиента). Не во внешние пути.
