## 1. СУВЕРЕНИТЕТ ДАННЫХ (Data Sovereignty)

Мы не верим в "облачное рабство". Ваши данные — ваша крепость.

- **Полный экспорт:** Любые данные доступны для выгрузки в CSV/JSON за 10 секунд через интерфейс Синдиката.
- **Право на забвение:** Полное стирание данных по запросу (GDPR/152-ФЗ). Никаких следов в бэкапах через 30 дней.
- **Self-Host:** При желании вы можете форкнуть репозиторий и уйти на свой сервер с вашими данными.

## 2. ЧТО МЫ СОБИРАЕМ (И ЗАЧЕМ)

### 2.1 Telegram (через @oneSitePlsBot)
- **ID пользователя, username, имя** — для аутентификации и персонализации.
- **Данные Mini App** — контекст использования (startapp параметры).
- **История платежей** — только если вы совершаете покупки (через Telegram Payments).

### 2.2 Supabase (PostgreSQL)
- **Профиль:** username, metadata (referrer codes, настройки).
- **Данные складов:** CSV импорты, структура товаров (только если вы их добавляете).
- **Логи действий:** JSON-логи для аудита (кто, что, когда изменил).

### 2.3 GitHub (для Supervibe Studio)
- **Токен доступа:** Только если вы используете функции создания PR через бота.
- **Репозитории:** Метаданные репо, которые вы выбираете для работы.

## 3. ЧТО МЫ НЕ ДЕЛАЕМ

- ❌ **Не продаем данные:** Никаких третьих лиц для маркетинга.
- ❌ **Не сливаем API ключи:** Ваши ключи WB/Ozon хранятся только в вашем ENV (если self-hosted) или зашифрованы.
- ❌ **Не шпионим:** Нет трекеров аналитики (Google Analytics, метрики), только системные логи Supabase.

## 4. ХРАНЕНИЕ И БЕЗОПАСНОСТЬ

- **Шифрование:** Все соединения TLS 1.3. База данных шифруется at-rest.
- **Доступ:** ENV переменные с секретами недоступны даже администраторам (только хеши).
- **Локация:** Серверы Supabase в EU (Frankfurt) — соответствие GDPR.

## 5. ВАШИ ПРАВА (152-ФЗ + GDPR)

Вы имеете право:
1. **Узнать** — какие данные у нас есть (запрос через @salavey13).
2. **Исправить** — обновить профиль через бота.
3. **Удалить** — полное забвение (right to be forgotten).
4. **Экспортировать** — получить все свои данные в машиночитаемом виде.
5. **Отозвать согласие** — на обработку (с удалением аккаунта).

## 6. COOKIES И ЛОКАЛЬНОЕ ХРАНИЛИЩЕ

- **Cookies:** Только технические (session_id, theme preference).
- **LocalStorage:** Кэш данных для офлайн-работы (не синхронизируется без вашего ведома).

## 7. ТРЕТЬИ СТОРОНЫ

- **GitHub** (опционально) — только если вы используете интеграцию.
