Step 3: Python Bridge Script

This script runs locally, receives Telegram messages, passes them to claude.cmd, and sends the response back.

Install Python dependencies in PowerShell:

pip install fastapi uvicorn requests

Create claude_telegram.py in your project folder:

# claude_telegram.py
import os
import asyncio
import requests
from fastapi import FastAPI, Request

app = FastAPI()

TELEGRAM_TOKEN = os.getenv("TELEGRAM_TOKEN", "token is here")
TELEGRAM_API_URL = f"https://api.telegram.org/bot{TELEGRAM_TOKEN}"

# Security: Only respond to these chat IDs
ALLOWED_CHAT_IDS = [413553377, 341729406]

async def run_claude_cli(prompt: str) -> str:
    """Runs the claude CLI tool asynchronously and returns the output."""
    try:
        process = await asyncio.create_subprocess_exec(
            "claude.cmd", "-p", prompt,
            stdout=asyncio.subprocess.PIPE,
            stderr=asyncio.subprocess.PIPE
        )
        stdout, stderr = await process.communicate()
        
        stdout_text = stdout.decode().strip()
        stderr_text = stderr.decode().strip()
        
        if process.returncode != 0:
            return f"⚠️ CLI Error:\n{stdout_text}\n{stderr_text}"
            
        return stdout_text
        
    except Exception as e:
        return f"❌ Server error: {str(e)}"

@app.post("/api/telegramWebhook")
async def telegram_webhook(request: Request):
    data = await request.json()
    
    if "message" in data and "text" in data["message"]:
        chat_id = data["message"]["chat"]["id"]
        user_text = data["message"]["text"]
        
        if chat_id not in ALLOWED_CHAT_IDS:
            print(f"Ignored message from unauthorized chat_id: {chat_id}")
            return {"ok": True}

        if user_text.startswith('/'):
            return {"ok": True}

        claude_reply = await run_claude_cli(user_text)
        
        if len(claude_reply) > 3900:
            claude_reply = claude_reply[:3900] + "... \n\n(Message truncated)"

        requests.post(f"{TELEGRAM_API_URL}/sendMessage", json={
            "chat_id": chat_id,
            "text": claude_reply
        })
        
    return {"ok": True}

if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8000)

Step 4: Expose Local Server (Tunnel)

Run in Git Bash using LocalTunnel (No signup required):

npx lt --port 8000 --subdomain car-test-local-$(date +%s)

Copy the HTTPS URL provided (e.g., https://car-test-local-...loca.lt)

Step 5: Connect Telegram Webhook

Run this in Git Bash, replacing <TUNNEL_URL> with the URL from Step 4:

npx tsx scripts/set-local-webhook.ts <TUNNEL_URL>

Example:

npx tsx scripts/set-local-webhook.ts https://car-test-local-1781134495.loca.lt

✅ If successful, you can now chat with your bot on Telegram!

Step 6: Running the Bot (Daily Workflow)

You need three terminals open:

  1. Git Bash (Tunnel): npx lt --port 8000 ...
  2. PowerShell (Python Server): python claude_telegram.py
  3. Git Bash (Webhook setter): Run the npx tsx scripts/set-local-webhook.ts command whenever your tunnel restarts and the URL changes.

🧠 Boss Mode Prompts & Notes

Key directives for the Claude Code CLI to self-orchestrate:

  • Supabase Integration: Check public.cars for bike specs/pricing. Only care about license categories A, B (includes M), or no license — ignore C/C1.
  • Cron Jobs: Create a Supabase skill to schedule cron jobs that trigger Claude to send project insights and enhancement proposals autonomously.
  • Subagent Orchestration: Spawn multiple subagents — create a plan for each as tasks in Supabase (supaplan). Merge and resolve conflicts on the fly; no PRs needed.
  • Boss Mode: Don't just execute — plan ahead. Create polishing steps, use "cheatcodes" to pick tasks from supaplan, reduce file-level merge conflicts by not storing shared state in repo.
  • Installer-Creator Skill: One single file approach: file starts with instructions → agent extracts the first code block as an install script → stops reading → runs the script → script unpacks the rest of the file into proper skill folders. Enhance repotxtfetcher with a button that prepends the extraction instruction + unpacker script automatically.

🔄 SUPALOOP

The autonomous execution loop:

  1. PLAN → Ask Claude to create supaplan tasks for a feature
  2. PICK → Claude uses cheatcodes to pull the next task from supaplan
  3. EXECUTE → Spawn subagent, merge on completion, no PR friction
  4. POLISH → Auto-generate refinement tasks back into supaplan
  5. REPEAT → Loop back to PICK — continuous, self-feeding cycle

⚡ Cron-triggered SUPALOOP: Supabase cron pings Claude on schedule → Claude picks pending supaplan tasks → executes → pushes insights back to you. The bot never stops improving the project.
