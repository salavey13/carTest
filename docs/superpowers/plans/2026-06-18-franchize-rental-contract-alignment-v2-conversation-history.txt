● Enhanced Web Link Handling ✓

  The contract-draft page now handles multiple scenarios:

  States Supported:

  1. Pending (owner viewing draft):
    - Shows draft details
    - Approve/Decline buttons for crew owner
  2. Approved (after contract creation):
    - Shows download link for DOCX
    - QR code for client to scan and fill final data
    - QR points back to same page for data entry
  3. Declined:
    - Shows declined message with instructions

  QR Code Feature:

  - Displayed when contract is approved
  - Points to: /franchize/{slug}/contract-draft/{rentalId}
  - Client can scan to:
    - View contract details
    - Fill final odometer/damage reports
    - Complete post-rental data

  Post-Rental Integration:

  - RentalReturnPanel shown when rental is active/completed
  - Allows filling:
    - Odometer at return
    - Fuel/battery level at return
    - Damage notes at return

  User Flow:

  1. User fills draft → owner approves → DOCX created
  2. Owner sees QR code → shows to user
