# SupaPlan Mobile UX Guide

## Цель мобильного режима
Сделать `/supaplan` похожим на плотный мобильный дашборд (в духе Supabase):
- короткие KPI-блоки,
- компактные карточки,
- переключение статусов без перегруза.

## Паттерны
1. **Тумблер «Рабочий поток / Архив»** всегда в верхней зоне списка.
2. **Статусы на мобильном** показываются как горизонтальные «чипы» с количеством задач.
3. **Карточка задачи**: заголовок + 2-3 метаполя + CTA без лишнего текста.
4. **События**: ограниченная высота списка, чтобы не съедать весь экран.

## Ограничения
- Не делать длинные многострочные CTA.
- Не использовать английские заголовки в видимом интерфейсе страницы.
- Не добавлять тяжёлые анимации в списках карточек.

## Быстрый smoke-check
- Ширина 390px: статусы доступны через горизонтальный скролл чипов.
- Ширина 768px+: появляется колонная группировка по статусам.
- Фильтр + поиск не ломают отображение «Архива».
