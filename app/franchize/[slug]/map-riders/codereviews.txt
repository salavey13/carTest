###  Разбор по пунктам (Senior Perspective)

#### 1. "7 Bounded Contexts" (Архитектура)
Ревьювер предлагает DDD (Domain Driven Design).
*   **Мнение:** Это перебор для фичи. Не нужно городить слои "Domain", "Application", "Infrastructure".
*   **Решение:** Тебе нужно просто **Container/Presentational** разделение.
    *   `useMapRiders` (Custom Hook) — тут вся грязь: API, сокеты, стейт, эффекты.
    *   `MapRidersView` (Component) — тут только JSX и пропсы.

#### 2. "useEffect Hell" (Эффекты)
Он абсолютно прав. У тебя куча зависимостей, которые триггерят друг друга.
*   **Опасный момент:** `setSessionId` -> `fetchSnapshot`. Это классическая race condition. Ты ждешь ответа от сервера, чтобы обновить UI, а реалтайм уже пнул обновление.
*   **Фикс:** Либо optimistic updates (сразу обновляем UI, потом отправляем запрос), либо строгая блокировка обновлений, пока ждем ответ.

#### 3. "Performance" (mapPoints)
*   **Вердикт:** Да, он прав. Ты собираешь массив `mapPoints` целиком при каждом обновлении. Если у тебя 100 маркеров, и один обновил координаты — React пересобирает массив из 100 объектов заново.
*   **Фикс:** Разбить на `useMemo`. Но лучше вообще хранить структуру в стейте, а не пересобирать массив.

#### 4. "Window Events"
*   **Вердикт:** `window.addEventListener("live-rider-update", ...)` — это грязный хак.
*   **Фикс:** Убери это. Если есть Supabase Realtime — юзай только его. Если нужно общаться между компонентами — юзай `EventTarget` или простой Zustand/PubSub.

#### ✅ Step 1: Extract the Engine Hook
Create `hooks/use-map-riders-engine.ts` (or merge into your existing `useMapRidersContext`):
```ts
// hooks/use-map-riders-engine.ts
import { useEffect, useRef, useCallback } from 'react';
import { getSupabaseBrowserClient } from '@/lib/supabase-browser';
import { mapRidersReducer, initialMapRidersState } from '@/lib/map-riders-reducer';

export function useMapRidersEngine(crewSlug: string) {
  const [state, dispatch] = React.useReducer(mapRidersReducer, initialMapRidersState);
  const supabase = useRef(getSupabaseBrowserClient());
  const channelRef = useRef<any>(null);
  const lastFetchRef = useRef(0);

  // 1. Initial snapshot + periodic refresh
  useEffect(() => {
    const fetchSnapshot = async () => {
      if (Date.now() - lastFetchRef.current < 2000) return; // debounce
      lastFetchRef.current = Date.now();
      try {
        const res = await fetch(`/api/map-riders/overview?slug=${crewSlug}`);
        const json = await res.json();
        if (json.success) dispatch({ type: 'snapshot/loaded', payload: json.data, selfUserId: /* dbUser */ });
      } catch {}
    };
    fetchSnapshot();
    const interval = setInterval(fetchSnapshot, 8000);
    return () => clearInterval(interval);
  }, [crewSlug]);

  // 2. Realtime subscription (REPLACE window events)
  useEffect(() => {
    const channel = supabase.current.channel(`map-riders:${crewSlug}`);
    channel.on('broadcast', { event: 'rider:move' }, ({ payload }) => {
      dispatch({ type: 'rider/moved', payload });
    });
    channel.subscribe();
    channelRef.current = channel;
    return () => { supabase.current.removeChannel(channel); };
  }, [crewSlug]);

  // 3. Stale eviction timer
  useEffect(() => {
    const tick = () => dispatch({ type: 'eviction/tick' });
    const interval = setInterval(tick, 5000);
    return () => clearInterval(interval);
  }, []);

  // 4. Actions (optimistic + network)
  const startShare = useCallback(async (params) => { /* fetch + dispatch share/started */ }, []);
  const stopShare = useCallback(async () => { /* fetch + dispatch share/stopped */ }, []);

  return { state, dispatch, actions: { startShare, stopShare, /* ... */ } };
}
```
